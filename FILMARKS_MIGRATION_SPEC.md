# CINEMA ROULETTE — データソース移行仕様書（MOVIE WALKER → Filmarks）

最終更新: 2026年5月21日
対象: `scripts/scrape.py` の全面改修（および `index.html` の軽微な調整）

---

## 0. この移行のゴール（なぜやるか）

現行は MOVIE WALKER PRESS をスクレイピングしているが、以下の問題があった。

- 見放題 / レンタル / 購入の区別ができない（「見放題とは限らない」と割り切っていた）
- Netflix のあらすじが空ばかり
- 作品数が少なく見える（Netflix 424 など）

データソースを **Filmarks** に切り替えることで、これらをすべて解決する。

- Filmarks は各作品に「見放題 / レンタル / 購入」ラベルが付くため、**見放題だけに絞れる**
- あらすじ・サムネ・ジャンル・公開年・評価が一覧ページから取得できる
- 各サービスの正式な視聴URL（Netflix作品ページ / Amazon作品ページ）が取れる
- 作品数が桁違いに多い（少なく見える問題が消える）

---

## 1. 取得対象URL

一覧ページはページ番号を `?page=N` で素直に送れる（無限スクロールではない。検証済み）。

- Netflix: `https://filmarks.com/list/vod/netflix?page=N`
- Prime Video: `https://filmarks.com/list/vod/prime_video?page=N`

`N` は 1 始まり。1ページあたり概ね 20〜25 作品。

**重要: 必ず `/list/vod/...`（映画）のURLを使う。** `/list-drama/vod/...`（ドラマ）や `/list-anime/vod/...`（アニメ）は使わない。映画だけが対象。これにより「映画とTV番組が混在する」問題が最初から発生しない。

---

## 2. 取得方針（件数とフィルタ）

### Netflix
- **見放題ラベルが付いた作品のみ採用**（実際には Netflix は全作品が「見放題」表示なので、結果的にほぼ全件になる）
- 全ページを巡回して全件取得（約150ページ想定。Netflix 映画は約3,400件）
- レンタル/購入のみの作品があれば除外（基本的に存在しないはず）

### Prime Video
- **見放題ラベルが付いた作品のみ採用**
- 「レンタルのみ」「購入のみ」の作品は**除外**する
- **人気順（おすすめ順＝デフォルト表示順）の上位から巡回し、採用した（＝見放題の）作品が 2,500 本に達したら打ち切る**
  - 上限の定数: `PRIME_TARGET_COUNT = 2500`
  - 全21,000件超を巡回すると800ページ超で運用が破綻するため、上位で打ち切る方針
  - 見放題比率次第だが、概ね 150〜250 ページ程度の巡回で 2,500 本に届く見込み

### 見放題判定のロジック
各作品ブロックには配信形態を示すラベルテキストが含まれる（例: 「見放題」「レンタル」「購入」）。
- そのブロック内に **「見放題」という文字列が含まれているか**で判定する
- 含まれていれば採用、含まれていなければスキップ
- Netflix・Prime とも同じ判定を通す

## 3. 各作品から取得するデータ

一覧ページの各作品ブロックから、以下を取得する。

| フィールド | 取得元 | 備考 |
|-----------|--------|------|
| `id` | 作品ページURL `/movies/数字` の数字 | 例: `/movies/120747` → `"120747"` |
| `title` | 作品タイトル見出し | |
| `url` | `https://filmarks.com/movies/{id}` | Filmarks作品ページ（クエリは付けない） |
| `thumb` | サムネ画像のURL | `cloudfront.net/.../_.jpg` 形式。無ければ空文字 |
| `year` | 「上映日：YYYY年MM月DD日」から西暦4桁を抽出 | 取れなければ null |
| `platform` | `"netflix"` または `"prime"` | 巡回中のサービスで決まる |
| `genres` | ジャンルのリンクテキスト（複数） | 例: `["クライム","ミステリー"]` |
| `synopsis` | 「あらすじ」セクションの本文 | 末尾の「>>続きを読む」は除く。無ければ空文字 |
| `watch_url` | サービス別の視聴URL（下記） | |

### watch_url の作り方（重要）

各作品ブロックには配信サービスへの誘導リンクがある。

- **Netflix**: `https://www.netflix.com/jp/title/数字` 形式のリンクがそのまま取れる。これを `watch_url` に使う。
- **Prime**: `https://www.amazon.co.jp/dp/XXXXXXXXXX?tag=filmarks_web-22` 形式のリンクが取れる。
  - **`?tag=filmarks_web-22` などのクエリパラメータは必ず削除し、`https://www.amazon.co.jp/dp/XXXXXXXXXX` だけにする**
  - （理由: これは Filmarks のアフィリエイトタグであり、他者のIDをそのまま転載しない）
- どちらも取れない場合は、フォールバックとして Filmarks 作品ページ `url` を使う。

### 取得してはいけないもの（重要）

- **ユーザーレビュー本文（「おすすめの感想・評価」セクション）は一切取得・保存しない。**
  - 他人の著作物であり、転載は不可
  - 機能上も不要
- 監督・脚本・出演者は取得しなくてよい（任意。あれば将来使えるが、今回のサイト機能には不要）

## 5. 巡回スケジュール / ブロック対策（運用設計）

現行の「1日1サービス・4日サイクル」を踏襲しつつ、Filmarks の規模に合わせて調整する。

### 基本方針
- **1日1サービスのみ取得**（連続巡回でのブロックを避ける現行の知見を維持）
- 既存データは保持する（その日取得しないサービスは前回データが残る → サイトには常に全作品が揃う）

### サイクル設計（提案）
Netflix（約150ページ）と Prime（上限2,500本／150〜250ページ）はどちらも1日で取り切るにはやや重い。安全側に倒し、各サービスを2分割して4日サイクルにする。

| 位相 (phase) | 取得内容 |
|------|----------|
| 0 | Netflix 前半（page 1〜終わりの半分） |
| 1 | Netflix 後半（残り） |
| 2 | Prime 前半（人気順 page 1〜、採用1,250本まで） |
| 3 | Prime 後半（続き、採用2,500本到達 or ページ尽きるまで） |

- `phase = today.toordinal() % 4` で当日の担当を決める
- ただし「採用本数で打ち切る Prime」をページ範囲で2分割するのは状態管理が複雑になる。
  **実装が難しければ次の簡易版でよい**:

### サイクル設計（簡易版・実装が楽ならこちら推奨）

| 位相 (phase) | 取得内容 |
|------|----------|
| 0, 1 | Netflix 全件（1日で全ページ巡回。重ければ2日に分割可） |
| 2, 3 | Prime 見放題を上位から 2,500 本まで（1日で巡回） |

- まずは「phase 偶数=Netflix全件 / phase 奇数=Prime（2,500本上限）」のように、**1サービスを1日で取り切る**形で実装してよい。
- 1日で取り切ると重い・ブロックされる場合のみ、ページ範囲分割を導入する。
- **どちらにせよ「1日に1サービスだけ」は厳守。**

### 主要な設定値（scrape.py 冒頭・現行を流用＆追加）
```python
SLEEP_LIST = 1.5            # 一覧ページ間の待機（秒）。重ければ 2.0〜3.0 に上げる
TIMEOUT = 60
MAX_RETRY = 3
FIRST_PAGE_RETRY = 6
STOP_AFTER_404 = 3          # 連続404でそのサービスの巡回を打ち切り
HARD_PAGE_CAP = 900         # 安全上限（Prime対策で現行800から引き上げ）
PRIME_TARGET_COUNT = 2500   # ★新規: Prime見放題の採用上限
CYCLE_LEN = 4
```

## 6. index.html 側の対応

データ構造はほぼ維持なので変更は最小限。

1. **視聴リンクを `watch_url` に切り替える**
   - 現行は「Netflix検索ページ / Amazon検索ページに作品名を投げる」方式。
   - 今後は各作品の `watch_url`（Netflix作品ページ / Amazon作品ページ直リンク）を使う。
   - `watch_url` が無い古いデータが混ざる可能性に備え、無ければ現行の検索URL方式にフォールバックしてもよい。

2. **「作品情報を見る」リンク**は引き続き作品の `url`（Filmarks作品ページ）を指す。

3. **フッターの注記を更新**
   - 現行: 「Prime Videoは見放題・レンタル・購入を区別していない…」という注記がある。
   - **見放題のみを掲載する方針になったため、この注記は実態に合わない。** 「見放題作品のみ掲載。配信状況は変動するため最終確認は各サービスで」といった主旨に差し替える。
   - データ出典の表記も「MOVIE WALKER PRESS」→「Filmarks」に変更。

4. 作品数の合計表記は既に削除済み（変更不要）。

## 7. 検証時のチェックポイント（実装後に確認すること）

- Netflix・Prime それぞれ、取得件数が妥当か（Netflix 数千件、Prime 2,500件前後で打ち切られているか）
- Prime に「レンタルのみ/購入のみ」の作品が混入していないか（サンプルを目視）
- `watch_url` が正しい形式か（Amazon は `?tag=...` が消えているか）
- あらすじがちゃんと入っているか（Netflix も埋まるはず）
- レビュー本文を取得していないこと
- 既存データ保持ロジックが効いているか（その日取得しないサービスのデータが残るか）

---

## 8. 注意・マナー

- 一覧ページのみ巡回する（個別作品ページは叩かない）。現行同様、軽量に保つ。
- `SLEEP_LIST` を入れて過度な負荷をかけない。
- robots/利用規約に反する使い方はしない。取得は基本情報のみ、レビュー等の著作物は保存しない。
- User-Agent は良識的なものを設定する。
